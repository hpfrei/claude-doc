Lifecycle hooks in .claude/settings.local.json
Hooks fire automatically at lifecycle points — executed by the harness, not by Claude. Live in settings.json (user / project / local scope).

Handler types: command (shell, JSON on stdin, exit 0 = allow / 2 = block), prompt (single-turn LLM eval), agent (sub-agent with Read/Grep/Glob). Synchronous by default; set async: true for fire-and-forget.

JSON output can block tools, modify inputs, inject context, or halt Claude.
PreToolUse → Tool Executes → PostToolUse → Claude Responds → Stop
Notification — Fires on notification events SubagentStop — Fires when sub-agent completes
PreToolUse hook Runs before a tool executes — can block or modify ▶

Description

Fires before any tool call. Can approve, block, or modify it based on exit code and stdout.

Environment Variables

VariableDescription
CLAUDE_TOOL_NAMETool name (e.g. Bash, Write)
CLAUDE_TOOL_INPUTJSON string of tool input
CLAUDE_SESSION_IDSession identifier

Exit Codes

  • 0 — Allow (stdout shown to Claude as context)
  • 2 — Block (stdout is the reason)
  • Other — Hook error; tool proceeds with warning

Use Cases

  • Prevent writes to protected files
  • Enforce naming conventions
  • Gate destructive Bash commands
  • Audit logging

Frontmatter reference

---
name: code-reviewer        # required
description: |             # required
  Reviews code for quality, security,
  and style.
model: sonnet              # sonnet/opus/haiku
tools: [Read, Grep, Glob, Bash]
disallowedTools: [Write, Edit]
maxTurns: 20
effort: high
background: false          # true = concurrent

  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./validate.sh"
---

You are a code reviewer...

Fresh context per invocation. Only a summary returns to parent. Invoke via Agent tool, @mention, or claude --agent name.

New Hook

Handler types

# command — shell, JSON on stdin
# prompt  — single-turn LLM eval
# agent   — sub-agent with Read/Grep/Glob

Exit codes (command)

exit 0  # success, parse JSON stdout
exit 2  # block tool (PreToolUse)
other   # error, continue

JSON output (exit 0)

{
  "hookSpecificOutput": {
    "permissionDecision": "deny",
    "updatedInput": {"command": "..."},
    "additionalContext": "injected text"
  }
}

Examples

# Auto-format after writes:
jq -r '.tool_input.file_path' |
  { read -r f; prettier --write "$f"; }

# Block dangerous commands:
jq -r '.tool_input.command' |
  grep -q 'rm -rf' &&
  echo '{"decision":"block"}'

Matcher: regex on tool name. Omit or * to match all.

Lifecycle hooks in .claude/settings.local.json
Hooks are user-configured shell commands that execute automatically in response to Claude Code lifecycle events. Defined in settings.json, they are run by the harness — not by Claude.
PreToolUse → Tool Executes → PostToolUse → Claude Responds → Stop
Notification — Fires on notification events SubagentStop — Fires when sub-agent completes
PreToolUse hook Runs before a tool executes — can block or modify ▶

Description

Fires before any tool call. Can approve, block, or modify it based on exit code and stdout.

Environment Variables

VariableDescription
CLAUDE_TOOL_NAMETool name (e.g. Bash, Write)
CLAUDE_TOOL_INPUTJSON string of tool input
CLAUDE_SESSION_IDSession identifier

Exit Codes

  • 0 — Allow (stdout shown to Claude as context)
  • 2 — Block (stdout is the reason)
  • Other — Hook error; tool proceeds with warning

Use Cases

  • Prevent writes to protected files
  • Enforce naming conventions
  • Gate destructive Bash commands
  • Audit logging

Configuration Reference
Hooks live in settings.json at three scopes: Project (.claude/settings.json or .claude/settings.local.json), User (~/.claude/settings.json), Global (~/.claude/settings.global.json).
Hook Object Schema config Structure of a hook entry in settings.json ▶ 
{
  "hooks": {
    "<HookType>": [
      {
        "matcher": "<ToolName>",   // optional: filter by tool
        "command": "<shell cmd>",   // required
        "timeout": 10000            // optional: ms
      }
    ]
  }
}

Fields

FieldTypeReqDescription
matcherstringoptTool name filter (PreToolUse/PostToolUse only). Omit for all tools.
